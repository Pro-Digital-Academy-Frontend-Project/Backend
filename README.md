## 📍협업규칙

### 1. 브랜치 규칙

feat/#이슈번호-기능요약

ex) feat/#12-login

**브랜치 새로 만들 때 'dev' 브랜치에 만들기**

### 2. Commit Message

Feat(#이슈번호): 작업내용

ex) Feat(#12): 로그인 기능 추가

- feat : 새로운 기능 추가
- fix : 버그 수정, 기능 수정
- ci/cd : 배포 관련
- docs : 문서 수정
- refactor : 코드 리팩토링 (변수명 수정 등)
- setting : 개발 환경 세팅
- test : 테스트 코드, 리팩토링 테스트 코드 추가
- remove : 파일 또는 코드, 리소스 제거
- resource : 이미지 리소스, prefab 등의 코드와 상관없는 리소스 추가

### 3. Pull Request

#이슈번호 Feat: 작업내용

ex) #12 Feat: 로그인 기능 추가
